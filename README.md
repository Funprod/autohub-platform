# AutoHub Platform

Единая платформа для автодилерской сети: CRM для дилерских центров + модуль контакт-центра.
Объединяет управление продажами, воронку сделок и мониторинг работы операторов колл-центра в одной системе.

## Функциональность (в разработке)

- [ ] Авторизация (JWT, OAuth Google, восстановление пароля)
- [ ] Админ-панель (управление пользователями и ролями)
- [ ] Дашборд аналитики дилерского центра
- [ ] Воронка продаж
- [ ] Конструктор обращений на тест-драйв
- [ ] Мониторинг звонков колл-центра в реальном времени
- [ ] Внутренние чаты (WebSocket)
- [ ] AI-помощник

## Стек

**Frontend:** Next.js, React 18+, TypeScript
**State management:** Redux Toolkit, RTK Query
**UI:** MUI, Styled Components
**Формы:** React Hook Form, Zod
**Тестирование:** Jest, React Testing Library, Storybook
**Архитектура:** Feature-Sliced Design (FSD)

## Архитектура проекта

Проект построен по методологии **Feature-Sliced Design**:

\`\`\`
src/
app/ — инициализация приложения, провайдеры, роутинг
processes/ — сквозные бизнес-процессы
widgets/ — композитные блоки интерфейса
features/ — пользовательские сценарии/действия
entities/ — бизнес-сущности (user, car, deal и т.д.)
shared/ — переиспользуемые утилиты, UI-kit, конфиги
\`\`\`

## Запуск проекта

\`\`\`bash
npm install
npm run dev # запуск дев-сервера
npm run test # unit-тесты
npm run lint # линтер
npm run format # форматирование Prettier
\`\`\`

## Архитектурные решения (ADR)

**Store и слои FSD:** `app/store/store.ts` — единственный файл в проекте,
которому разрешён импорт из entities/features напрямую (нарушение общего
правила слоёв). Это осознанное исключение, задокументированное в ESLint-конфиге.
